1. การตรวจจับ: เหตุการณ์

คืออะไร: ระบบ VMS (Video Management System) หลัก ตรวจจับเหตุการณ์ที่กำหนดไว้ล่วงหน้า เช่น การเคลื่อนไหวในพื้นที่หวงห้ามหลังเวลาทำการ

การทำงาน: ระบบจะส่งข้อมูลเหตุการณ์ (ID กล้อง, เวลา, ประเภทเหตุการณ์) ไปยัง URL ของ n8n webhook ทันที

2. AI และกฎ: สมองกล (n8n)

คืออะไร: n8n workflow รับข้อมูลและทำหน้าที่เป็นเครื่องมือจัดการกฎอัจฉริยะ

การทำงาน: ประมวลผลตรรกะที่เรากำหนดเอง เช่น `ถ้าเหตุการณ์คือ 'การเคลื่อนไหว' และเวลาคือหลัง '22:00 น.' ให้ส่งการแจ้งเตือน`

3. การควบคุม: ศูนย์กลาง (Django)

คืออะไร: web application Django ที่ปลอดภัยทำหน้าที่เป็นศูนย์ควบคุมหลักสำหรับผู้ใช้งาน

การทำงาน: รับ API call จาก n8n เพื่อบันทึกเหตุการณ์ลงในฐานข้อมูล และเตรียมแจ้งเตือนผู้ใช้งานแบบสด

4. การแจ้งเตือน: การดำเนินการ

เมื่อตรงตามกฎ n8n และ Django จะดำเนินการแจ้งเตือน 3 รูปแบบพร้อมกัน

แจ้งเตือนบนหน้าจอทันที

web application ใช้ WebSockets เพื่อส่งการแจ้งเตือนแบบเรียลไทม์ไปยัง "จอแสดงผลหลัก" บนหน้าจอของผู้ใช้งาน

แจ้งเตือนทางอีเมล

n8n จะสร้างและส่งรายงานเหตุการณ์โดยละเอียดไปยังรายชื่อผู้รับอีเมลด้านความปลอดภัยที่กำหนดไว้โดยอัตโนมัติ

แจ้งเตือนผ่าน LINE

n8n เรียกใช้ LINE webservice API เพื่อส่งข้อความโดยตรงไปยังบุคคลหรือกลุ่มแชทที่ระบุไว้ทันที
