1

การตั้งค่าโครงสร้าง MikroTik VPN

สร้างการเชื่อมต่อที่ปลอดภัยและเข้ารหัสระหว่าง Remote Site ทั้งหมดกับศูนย์ควบคุมหลัก:

  1. เชื่อมต่อกับ MikroTik Router แต่ละตัวผ่าน application WinBox (IP เริ่มต้น: 192.168.88.1)
  2. ไปที่ IP → IPsec → Peers และตั้งค่า peer connection ใหม่ด้วย public IP address ของ router สำนักงานใหญ่
  3. ตั้งค่า shared key authentication ด้วยการเข้ารหัสแรงขั้นต่ำ 256-bit
  4. เปิดใช้งาน Phase 1 (IKE) และ Phase 2 (ESP) ด้วยการเข้ารหัส AES-256-GCM และการยืนยันตัวตน SHA-256
  5. ตั้งค่า NAT traversal สำหรับ Site ที่อยู่หลัง firewall/NAT device
  6. ทดสอบการเชื่อมต่อโดยการ ping IP LAN ของ NVR จาก Remote Site แต่ละแห่ง
  7. ใช้กลไก failover สำหรับการเชื่อมต่อซ้ำซ้อน
2

การแบ่งกลุ่ม Network และความปลอดภัย

ใช้ความปลอดภัย Network ที่แข็งแกร่งผ่านการแยก VLAN และ access control:

  1. สร้าง VLAN 10 เฉพาะสำหรับ surveillance camera บน PoE switch ทั้งหมด
  2. ตั้งค่า VLAN 20 สำหรับ management interface และการเข้าถึง NVR
  3. ใช้ inter-VLAN routing ด้วย access control list (ACL) ที่เข้มงวด
  4. บล็อกการเข้าถึง internet ของ camera ผ่าน firewall rule เพื่อป้องกันการโจมตีจากภายนอก
  5. ตั้งค่า Quality of Service (QoS) เพื่อให้ความสำคัญกับ video traffic มากกว่า network traffic อื่น
  6. ตั้งค่า network monitoring และ intrusion detection system
  7. ใช้ MAC address filtering เป็นชั้นความปลอดภัยเพิ่มเติม
3

การปรับ Network ให้เหมาะสมขั้นสูง

ปรับประสิทธิภาพ Network สำหรับการสตรีม video คุณภาพสูง:

  1. ตั้งค่า multicast routing สำหรับการกระจาย video ที่มีประสิทธิภาพ
  2. ตั้งค่า link aggregation (LACP) เพื่อเพิ่ม bandwidth
  3. ใช้ network time protocol (NTP) synchronization ข้าม device ทั้งหมด
  4. ตั้งค่า SNMP monitoring สำหรับการจัดการสุขภาพ network แบบเชิงรุก
  5. ตั้งค่า bandwidth monitoring และ alerting threshold

แนวทางปฏิบัติด้านความปลอดภัยที่ดี

เปลี่ยน credential เริ่มต้นเสมอ เปิดใช้งานการเข้ารหัส WPA3 สำหรับการเชื่อมต่อ wireless และอัปเดต firmware บน network device ทั้งหมดอย่างสม่ำเสมอ ใช้การยืนยันตัวตน VPN แบบ certificate สำหรับความปลอดภัยที่เพิ่มขึ้น

Phase 3: การตั้งค่า NVR และการรวม Camera

1

การเริ่มต้นระบบ NVR

ตั้งค่าระบบบันทึกหลักด้วยการตั้งค่า storage ที่เหมาะสม:

  1. เชื่อมต่อ NVR กับ management VLAN (VLAN 20) ด้วยการกำหนด static IP
  2. ติดตั้ง surveillance-grade HDD ในการตั้งค่า RAID 5 เพื่อความซ้ำซ้อน
  3. เปิดระบบและทำ initial setup wizard ให้เสร็จสิ้น
  4. เข้าถึง web interface ของ NVR และไปที่ Storage → Disk Management
  5. Format drive และตั้งค่า storage quota ต่อ camera channel
  6. ตั้งโหมดการบันทึกเป็น continuous + motion detection เพื่อการครอบคลุมที่เหมาะสม
  7. ตั้งค่า automatic disk cleanup policy เพื่อจัดการพื้นที่ storage
  8. เปิดใช้งาน SMART monitoring เพื่อการตรวจจับความผิดปกติของ drive แบบ predictive
2

การค้นหาและตั้งค่า Camera

รวม IP camera ทั้งหมดด้วยการตั้งค่า streaming ที่เหมาะสม:

  1. ไปที่ Camera → Remote Device → Auto Search เพื่อค้นหา ONVIF-compatible camera
  2. สำหรับการเพิ่มแบบ manual: ใส่ IP address ของ camera ด้วย credential ที่อัปเดตแล้ว (ไม่ใช้ค่าเริ่มต้น)
  3. ตั้งค่า dual-stream setting สำหรับ camera แต่ละตัว:
    • Main Stream: 4MP @ 15fps (H.265, 8Mbps) สำหรับการบันทึก
    • Sub-Stream: 720p @ 10fps (H.264, 2Mbps) สำหรับการติดตาม live
  4. เปิดใช้งานฟีเจอร์ smart recording (motion detection, line crossing, intrusion detection)
  5. ตั้งค่าเฉพาะ camera (exposure, WDR, noise reduction)
  6. ตั้งค่า privacy masking สำหรับพื้นที่ที่มีความอ่อนไหว
  7. ทดสอบ camera feed ทั้งหมดและตรวจสอบการทำงานของการบันทึก
3

ฟีเจอร์การบันทึกขั้นสูง

ตั้งค่าการบันทึกอัจฉริยะและการจัดการเหตุการณ์:

  1. ตั้งค่าโซน motion detection ด้วยระดับความไวที่ปรับได้
  2. ตั้งค่า event-triggered action (email alert, mobile notification)
  3. เปิดใช้งาน facial recognition และ license plate recognition ที่เหมาะสม
  4. ตั้งค่าการบันทึกสำรองไปยัง secondary storage หรือ cloud
  5. ตั้งค่า automatic video export สำหรับการจัดการเหตุการณ์

การคำนวณ Storage

สำหรับ camera 10 ตัวที่บันทึก 24/7 ในรูปแบบ H.265 ที่ 8Mbps: ประมาณ 8.6TB ต่อเดือน วางแผนสำหรับการเก็บ 3-6 เดือนด้วยความซ้ำซ้อน RAID

การตั้งค่าความปลอดภัย

เปลี่ยน password เริ่มต้นทั้งหมดทันที เปิดใช้งาน two-factor authentication ตั้งค่า IP whitelisting และตั้งค่าการเข้าถึง HTTPS ที่เข้ารหัสสำหรับการจัดการระยะไกล

Phase 4: Video Wall Controller และการตั้งค่า Display

1

การเชื่อมต่อ Hardware และการเริ่มต้น

สร้างการกระจาย video คุณภาพสูงข้าม display หลายตัว:

  1. เชื่อมต่อ output หลักของ NVR กับ input ของ Video Wall Controller ผ่าน HDMI 2.0 หรือ DisplayPort สำหรับสัญญาณ 4K
  2. เชื่อมต่อ monitor ทั้ง 5 ตัวกับ output ของ controller โดยใช้สายที่เหมาะสม (HDMI/DisplayPort)
  3. ตรวจสอบว่าการเชื่อมต่อทั้งหมดแน่นหนาและติดตั้งอย่างถูกต้อง
  4. เปิด device ตามลำดับ: Monitor → Controller → NVR
  5. ตรวจสอบ signal integrity และ resolution บน display ทั้งหมด
  6. ตั้งค่า display orientation และ bezel compensation
2

การตั้งค่า Layout ขั้นสูง

สร้าง layout การติดตามแบบ dynamic และมืออาชีพ:

  1. เข้าถึง interface ของ Vivotek controller ผ่าน web browser (IP เริ่มต้น: 192.168.0.100)
  2. สร้าง preset layout หลายแบบสำหรับสถานการณ์การติดตามที่แตกต่างกัน:
    • Overview Layout: กริด 3×3 ข้าม monitor 1-4, camera เดี่ยวบน monitor 5
    • Zone Focus: camera เฉพาะ site จัดกลุ่มตามสถานที่
    • Incident Response: camera ลำดับความสำคัญในมุมมองใหญ่พร้อม feed สนับสนุน
    • Night Mode: layout ที่เหมาะสมสำหรับการติดตามแสงน้อย
  3. ตั้งค่าการสลับ layout อัตโนมัติตามเวลาของวันหรือเหตุการณ์
  4. ตั้งค่าการหมุน camera สำหรับการครอบคลุมที่ครอบคลุม
  5. โปรแกรม hotkey และ remote control shortcut สำหรับการเปลี่ยน layout อย่างรวดเร็ว
  6. ทดสอบ layout ทั้งหมดและให้แน่ใจว่าการเปลี่ยนผ่านราบรื่น
3

การปรับ Display ให้เหมาะสมและการ Calibration

ให้แน่ใจว่าคุณภาพภาพเหมาะสมที่สุดข้าม monitor ทั้งหมด:

  1. ปรับการตั้งค่าสีและระดับความสว่างสำหรับการดูที่สม่ำเสมอ
  2. ตั้งค่า blue light filtering เพื่อความสบายของผู้ปฏิบัติงาน
  3. ตั้งค่าการจัดการไฟ monitor สำหรับประสิทธิภาพพลังงาน
  4. ใช้การป้องกัน anti-burn-in สำหรับองค์ประกอบคงที่
  5. ตั้งค่า on-screen display overlay (เวลา, ชื่อ camera, สถานะ)

แนวทางปฏิบัติที่ดีสำหรับ Layout

วาง camera สำคัญในพื้นที่ดูกลาง ใช้รูปแบบกี่สม่ำเสมอ และให้แน่ใจว่า operator control เข้าถึงได้ง่าย พิจารณาปัจจัย ergonomic สำหรับการติดตาม 24/7

Phase 5: Software การติดตามและ Remote Access

1

การตั้งค่า Client Software มืออาชีพ

ติดตั้ง software การติดตามและการจัดการที่ครอบคลุม:

  1. ดาวน์โหลดและติดตั้ง iVMS-4200 professional client จากเว็บไซต์อย่างเป็นทางการของ Hikvision
  2. เปิด application และไปที่ Device Management → Add Device
  3. ตั้งค่าการเชื่อมต่อ NVR ด้วยการสื่อสารที่เข้ารหัสที่เปิดใช้งาน
  4. สร้าง virtual video wall layout ที่ตรงกับการตั้งค่า monitor ทางกายภาพ
  5. ตั้งค่า user account ด้วย role-based access control (RBAC)
  6. ตั้งค่าการรายงานอัตโนมัติและการจัดการ log
  7. ทดสอบฟีเจอร์ client ทั้งหมดรวมถึง playback, export และ live viewing
2

การตั้งค่า Mobile และ Remote Access

เปิดใช้งานความสามารถการติดตามระยะไกลที่ปลอดภัย:

  1. ติดตั้งและตั้งค่า Hik-Connect mobile application บน device ที่ได้รับอนุญาต
  2. ตั้งค่า push notification สำหรับเหตุการณ์สำคัญและ system alert
  3. ตั้งค่าการเข้าถึงเว็บผ่าน secure HTTPS portal (port แนะนำ: 8443)
  4. ใช้การเข้าถึงเฉพาะ VPN สำหรับความปลอดภัยที่เพิ่มขึ้น (หลีกเลี่ยงการเปิดรับ internet โดยตรง)
  5. ตั้งค่า geofencing สำหรับ access control ตามสถานที่
  6. ตั้งค่าการดู offline และขั้นตอนการตอบสนองฉุกเฉิน
3

การรวมและ Automation

ใช้ automation การติดตามขั้นสูง:

  1. ตั้งค่า API integration สำหรับระบบความปลอดภัยของบุคคลที่สาม
  2. ตั้งค่าการรายงานเหตุการณ์อัตโนมัติ
  3. ใช้ intelligent video analytics และ behavioral detection
  4. ตั้งค่า protocol การติดตามสำรองและระบบซ้ำซ้อน

ฟีเจอร์ Mobile Access

เปิดใช้งาน PTZ control, two-way audio communication, instant replay และฟังก์ชัน emergency alert สำหรับความสามารถการติดตามระยะไกลที่ครอบคลุม

Phase 6: การทดสอบระบบและการปรับประสิทธิภาพ

1

การตรวจสอบระบบอย่างครอบคลุม

ทำการทดสอบอย่างละเอียดเพื่อให้แน่ใจว่าระบบเชื่อถือได้:

  1. ทำการทดสอบ failover โดยการถอดไฟจาก camera หนึ่งตัว → ยืนยันว่า NVR ยังคงบันทึกตัวอื่นต่อไป
  2. ทดสอบการเล่นกลับของ camera 10 ตัวพร้อมกันที่ความละเอียดสูงสุด → ตรวจสอบความล่าช้า
  3. ตรวจสอบการทำงานของ night vision และประสิทธิภาพการส่องสว่าง infrared
  4. ทดสอบความแม่นยำของ motion detection และอัตรา false positive
  5. ทำการทดสอบความเครียด network ด้วยการใช้ bandwidth สูงสุด
  6. ตรวจสอบขั้นตอนการสำรองและการกู้คืนด้วยสถานการณ์สูญเสียข้อมูลจำลอง
  7. ทดสอบ protocol การตอบสนองฉุกเฉินและระบบแจ้งเตือน
2

การปรับประสิทธิภาพและการปรับแต่ง

ปรับแต่งประสิทธิภาพระบบสำหรับการทำงานที่เหมาะสม:

  1. ปรับ frame rate ของ camera ตามความต้องการ bandwidth และ storage ที่มีอยู่
  2. เปิดใช้งานการบีบอัด H.265+ เพื่อลดความต้องการ storage ได้ถึง 50%
  3. ตั้งค่าการปรับ dynamic bitrate อัจฉริยะสำหรับความซับซ้อนของฉากที่แตกต่างกัน
  4. ตั้งค่าการจัดการ storage อัตโนมัติด้วย retention policy ที่ตั้งค่าได้
  5. ใช้ load balancing ข้าม storage drive หลายตัว
  6. ตั้งค่าการติดตามสุขภาพระบบด้วย predictive maintenance alert
3

เอกสารและคู่มือ

เตรียมเอกสารที่ครอบคลุมและการฝึกใช้

  1. สร้างเอกสารสถาปัตยกรรมระบบที่ละเอียด
  2. พัฒนา standard operating procedure (SOP) สำหรับการทำงานประจำวัน
  3. เตรียมคู่มือแก้ไขปัญหาและ protocol การตอบสนองฉุกเฉิน
  4. อบรม operator เกี่ยวกับฟีเจอร์ระบบและการบำรุงรักษา
  5. กำหนดตารางการบำรุงรักษาป้องกันและขั้นตอน

คู่มืออ้างอิงการแก้ไขปัญหาอย่างรวดเร็ว

  • ไม่มีการแสดง video: ตรวจสอบการเลือก input source ของ video wall controller และการเชื่อมต่อสาย
  • Camera offline: ตรวจสอบการตั้งค่า VLAN, PoE power supply และการเชื่อมต่อ network
  • ปัญหาการเชื่อมต่อ VPN: ตรวจสอบ MikroTik IPsec log และตรวจสอบพารามิเตอร์ phase 1/2
  • Storage เต็ม: ตรวจสอบ retention policy และเปิดใช้งานการทำความสะอาดอัตโนมัติ
  • คุณภาพภาพไม่ดี: ปรับการตั้งค่าการบีบอัดและตรวจสอบการจัดสรร bandwidth

รายการตรวจสอบความสำเร็จของการใช้งาน

  1. Camera ทั้งหมดบันทึกอย่างต่อเนื่องด้วยการตรวจสอบการสำรอง
  2. Video wall แสดง layout ที่ตั้งค่าทั้งหมดโดยไม่มีการขัดจังหวะ
  3. การเข้าถึงระยะไกลทำงานผ่าน mobile app และ web interface
  4. ระบบสำรองยืนยันการทำงานด้วยการทดสอบการกู้คืน
  5. password เริ่มต้นทั้งหมดถูกเปลี่ยนและมาตรการความปลอดภัยถูกใช้
  6. เมตริกประสิทธิภาพตรงตามข้อกำหนดที่ระบุ
  7. อบรม operator เสร็จสิ้นและส่งมอบคู่มือ
